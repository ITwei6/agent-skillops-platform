# 05. 前端实施计划

## 一、阶段定位

前端实现由窗口 B 负责。

当前窗口只定义前端大纲、页面、路由、组件、接口协作和验收标准。

## 二、第一阶段：工程初始化

目标：

- 初始化 Vue 3 + TypeScript + Vite 工程
- 接入 Element Plus
- 接入 Vue Router
- 接入 Pinia
- 封装 Axios
- 搭建基础 Layout

交付物：

```text
web/admin/
```

验收标准：

- `pnpm dev` 能启动
- 有登录页和基础布局
- 左侧菜单可跳转
- API request 封装完成

## 三、第二阶段：核心页面静态版

目标：

- 工作台
- 项目列表
- 经验列表
- 经验提交
- Skill 草稿列表
- Skill 草稿详情
- 审核队列
- 审核详情
- Skill 仓库

验收标准：

- 页面路由完整
- 组件布局合理
- 使用 mock 数据展示
- 无明显样式错乱

## 四、第三阶段：对接后端 API

目标：

- 对接项目接口
- 对接经验接口
- 对接 Skill 草稿接口
- 对接审核接口
- 对接 Skill 仓库接口

验收标准：

- 列表可分页
- 表单可提交
- 详情可查看
- 审核动作可调用
- 错误提示统一

## 五、第四阶段：体验增强

目标：

- MarkdownViewer
- SkillScorePanel
- CheckReportPanel
- ReviewTimeline
- SimilarSkillList
- 文件上传

验收标准：

- Skill 内容可读
- 检查报告清晰
- 审核历史完整
- 文件上传可用

## 六、第五阶段：权限和发布流程

目标：

- 菜单权限
- 路由权限
- 发布确认
- 废弃确认
- 使用反馈

验收标准：

- 不同角色看到不同菜单
- 无权限路由不可访问
- 危险操作二次确认
- 发布流程可闭环

## 七、前端验收标准

窗口 B 每次交付前需要确认：

```text
1. 页面是否符合文档路由
2. 组件是否复用
3. API 是否统一封装
4. 是否有 loading 状态
5. 是否有空状态
6. 是否有错误提示
7. 是否无权限保护
8. 是否无明显文字溢出
9. 是否和服务端接口文档一致
```

## 八、和后端窗口协作方式

如果后端接口未完成：

- 前端先使用 mock 数据
- API 模块保持真实函数名
- 不在页面中散落 mock 逻辑

当后端接口完成后：

- 替换 API 实现
- 页面不大改

如果接口字段和文档冲突：

- 先反馈当前设计窗口
- 不由前端窗口单方面改业务字段
