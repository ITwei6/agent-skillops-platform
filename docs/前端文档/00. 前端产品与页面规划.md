# 00. 前端产品与页面规划

## 一、文档目的

本文用于指导前端实现窗口进行页面设计和开发。

本项目采用前后端分离架构：

```text
Vue 前端
  -> gateway-service HTTP API
  -> C++ 微服务后端
```

当前窗口负责前端规划和规范，窗口 B 负责具体前端实现。

## 二、前端定位

前端不是宣传官网，也不是简单 CRUD 页面。

前端定位是：

```text
面向研发团队的 SkillOps 管理后台
```

重点是：

- 清晰展示待处理事项
- 快速提交经验记录
- 高效审核 Skill 草稿
- 可检索团队 Skill 仓库
- 清楚呈现风险、版本、审核历史和过期状态

## 三、目标用户

| 用户 | 页面重点 |
| --- | --- |
| 普通开发者 | 提交经验、查看本人提交、搜索 Skill |
| 技术审核人 | 审核队列、风险报告、相似 Skill、审核意见 |
| Skill 维护者 | 发布版本、废弃 Skill、处理过期提醒 |
| 管理员 | 用户、团队、项目、策略配置 |

## 四、页面总览

### 4.1 登录页

功能：

- 登录
- 展示系统名称
- 显示当前环境

一期可先做简单账号登录，不做复杂 SSO。

### 4.2 工作台

功能：

- 待审核数量
- 我的提交数量
- 最近发布 Skill
- 高风险草稿
- 可能过期 Skill
- 平台活动流

目标：

让用户打开系统后知道现在该处理什么。

### 4.3 项目管理

功能：

- 项目列表
- 创建项目
- 编辑项目
- 查看项目详情
- 查看项目下经验、Skill、文件

### 4.4 经验记录

页面：

- 经验列表
- 提交经验
- 经验详情

提交内容：

- 所属项目
- 标题
- 来源类型
- 经验总结
- 附件
- 可选 Skill 草稿

### 4.5 Skill 草稿

页面：

- 草稿列表
- 草稿详情
- 编辑草稿
- 自动检查报告
- 相似 Skill
- 提交审核

重点：

- 展示 `SKILL.md`
- 展示质量评分
- 展示风险等级
- 展示状态流转

### 4.6 审核中心

页面：

- 审核队列
- 审核详情

审核动作：

- 通过
- 驳回
- 要求修改
- 添加评论
- 标记重复
- 标记高风险

### 4.7 Skill 仓库

页面：

- Skill 搜索
- Skill 详情
- Skill 版本
- Skill 下载
- 使用反馈

展示重点：

- Skill 名称
- 类型
- 项目
- 最新版本
- 状态
- 评分
- 是否过期

### 4.8 文件产物

功能：

- 上传文件
- 查看文件详情
- 下载文件
- 查看文件关联对象

### 4.9 系统管理

页面：

- 用户管理
- 团队管理
- 角色管理
- 策略配置

一期可简化，只保留用户和团队基础管理。

## 五、页面优先级

### P0 第一阶段必须完成

- 登录页
- 工作台
- 项目列表/创建项目
- 经验提交/经验列表/经验详情
- Skill 草稿列表/详情
- 审核队列/审核详情
- Skill 仓库列表/详情

### P1 第二阶段完成

- 文件产物管理
- Skill 版本对比
- 使用反馈
- 过期 Skill 处理

### P2 第三阶段完成

- 统计分析
- 策略配置
- 批量审核
- 高级搜索
- 管理员控制台
