# 02. 路由与页面结构设计

## 一、路由设计原则

- 路由路径使用小写中划线。
- 详情页使用 ID 参数。
- 页面按业务模块分组。
- 审核、发布等动作不单独作为页面，放在详情页中操作。

## 二、路由总览

```text
/login
/
/dashboard
/projects
/projects/create
/projects/:projectId
/experiences
/experiences/create
/experiences/:experienceId
/skill-drafts
/skill-drafts/:draftId
/reviews
/reviews/:draftId
/skills
/skills/:skillId
/skills/:skillId/versions/:version
/artifacts
/artifacts/:artifactId
/system/users
/system/teams
```

## 三、页面说明

### 3.1 /login

登录页面。

表单字段：

- 邮箱
- 密码

一期可以先对接简化登录接口，后续再扩展。

### 3.2 /dashboard

工作台。

模块：

- 待审核卡片
- 我的提交卡片
- 高风险草稿卡片
- 可能过期 Skill 卡片
- 最近活动列表

### 3.3 /projects

项目列表。

字段：

- 项目名称
- 描述
- 仓库地址
- 状态
- 创建人
- 创建时间

操作：

- 新建
- 查看详情
- 编辑

### 3.4 /experiences

经验记录列表。

筛选项：

- 项目
- 来源类型
- 提交人
- 状态
- 时间范围

### 3.5 /experiences/create

经验提交页面。

表单：

- 项目
- 标题
- 来源类型
- 经验总结
- 附件
- Skill 草稿，可选

提交后跳转经验详情。

### 3.6 /experiences/:experienceId

经验详情页。

Tabs：

- 基础信息
- 原始内容
- 附件
- 分析任务
- 生成的 Skill 草稿

### 3.7 /skill-drafts

Skill 草稿列表。

筛选项：

- 项目
- 状态
- 风险等级
- 类型
- 创建人

### 3.8 /skill-drafts/:draftId

Skill 草稿详情。

布局建议：

```text
左侧：Skill 内容
右侧：质量评分、风险项、状态、操作按钮
底部：审核记录、相似 Skill、关联经验
```

操作：

- 编辑
- 提交审核
- 查看检查报告

### 3.9 /reviews

审核队列。

筛选项：

- 项目
- 风险等级
- 状态
- 提交人

### 3.10 /reviews/:draftId

审核详情。

必须展示：

- Skill 草稿内容
- 来源经验
- 自动检查报告
- 相似 Skill
- 审核历史

审核动作：

- 通过
- 驳回
- 要求修改
- 评论

### 3.11 /skills

Skill 仓库。

支持：

- 关键词搜索
- 项目筛选
- 类型筛选
- 状态筛选
- 风险等级筛选

### 3.12 /skills/:skillId

Skill 详情。

展示：

- 基础信息
- 最新版本
- description
- 类型
- 状态
- 评分
- 版本列表
- 使用反馈

### 3.13 /skills/:skillId/versions/:version

Skill 版本详情。

展示：

- `SKILL.md`
- references
- scripts
- changelog
- 发布信息
- 下载按钮

### 3.14 /artifacts

文件产物列表。

### 3.15 /system/users

用户管理。

### 3.16 /system/teams

团队管理。
