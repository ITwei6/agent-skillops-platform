# 03. 组件与交互规范

## 一、文档目的

本文定义前端公共组件和交互规范，避免窗口 B 在实现时每个页面风格不一致。

## 二、布局规范

使用后台系统布局：

```text
顶部：系统标题、当前用户、退出
左侧：导航菜单
主体：页面内容
```

不做营销式首页，不做大幅 Hero，不做装饰性卡片堆叠。

## 三、通用组件

### 3.1 PageHeader

用于页面标题区域。

包含：

- 标题
- 描述
- 主操作按钮

### 3.2 SearchBar

用于列表页筛选。

支持：

- 输入框
- 下拉选择
- 时间范围
- 查询按钮
- 重置按钮

### 3.3 DataTable

统一表格包装。

要求：

- 支持 loading
- 支持分页
- 支持空状态
- 操作列固定在右侧

### 3.4 StatusTag

用于显示状态。

状态颜色建议：

| 状态 | 颜色 |
| --- | --- |
| active/succeeded/approved | green |
| draft/queued | blue |
| running/submitted | cyan |
| needs_changes/possibly_outdated | orange |
| rejected/failed/blocked | red |
| deprecated/superseded | gray |

### 3.5 RiskTag

风险等级：

| 风险 | 颜色 |
| --- | --- |
| low | green |
| medium | orange |
| high | red |
| blocked | dark red |

### 3.6 MarkdownViewer

用于展示：

- `SKILL.md`
- 审核意见
- 经验总结
- changelog

要求：

- 支持代码块
- 支持复制
- 支持目录，可后续扩展

### 3.7 SkillScorePanel

展示质量评分：

- 总分
- 场景明确性
- 触发描述准确性
- 可复现性
- 来源可信度
- 安全性
- 维护性

### 3.8 CheckReportPanel

展示自动检查结果：

- 敏感信息检查
- 重复检查
- 范围检查
- 格式检查
- 危险命令检查

### 3.9 ReviewTimeline

展示审核历史：

- 提交审核
- 审核意见
- 要求修改
- 通过
- 驳回
- 发布

### 3.10 SimilarSkillList

展示相似 Skill：

- Skill 名称
- 相似度
- 类型
- 状态
- 差异摘要

## 四、交互规范

### 4.1 表单提交

- 提交前前端做基础必填校验。
- 提交按钮进入 loading。
- 成功后给出明确提示。
- 失败后显示后端返回的 `message`。

### 4.2 危险操作

以下操作必须二次确认：

- 驳回 Skill
- 发布 Skill
- 废弃 Skill
- 回滚版本
- 删除/禁用用户

### 4.3 审核操作

审核操作必须要求填写意见：

- 驳回：必填
- 要求修改：必填
- 通过：可选，但 high 风险必填

### 4.4 空状态

列表为空时不要只显示空白。

示例：

```text
暂无 Skill 草稿，提交一条经验后可生成草稿。
```

### 4.5 错误状态

接口错误应区分：

- 无权限
- 数据不存在
- 网络错误
- 服务异常

## 五、视觉风格

整体风格：

- 企业后台
- 信息密度适中
- 清晰、克制、稳定
- 避免花哨渐变和装饰图

建议：

- 主色使用蓝色或深青色
- 风险色使用红/橙
- 成功色使用绿
- 页面背景使用浅灰
- 内容区域使用白底
