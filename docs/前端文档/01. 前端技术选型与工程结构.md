# 01. 前端技术选型与工程结构

## 一、技术选型

推荐技术栈：

```text
Vue 3
TypeScript
Vite
Element Plus
Pinia
Vue Router
Axios
```

## 二、选择原因

### 2.1 Vue 3

适合快速构建企业后台系统，学习成本低，生态成熟。

### 2.2 TypeScript

接口字段较多，必须使用类型约束，避免前后端字段混乱。

### 2.3 Element Plus

后台系统常用组件完整：

- 表格
- 表单
- 弹窗
- Tabs
- Tree
- Upload
- Tag
- Badge
- Drawer
- Pagination

### 2.4 Pinia

用于保存：

- 当前用户
- token
- 权限
- 全局字典
- 项目选择状态

### 2.5 Axios

统一封装 HTTP 请求、错误处理和 token 注入。

## 四、前端边界

前端负责：

- 页面展示
- 表单校验
- 用户交互
- 调用后端 API
- 状态提示
- 路由权限

前端不负责：

- 业务状态最终判定
- 审核权限最终判定
- Skill 风险计算
- 版本发布合法性判断
- 文件实际存储

所有关键业务规则以后端为准。

## 五、环境配置

建议使用：

```text
.env.development
.env.production
```

核心变量：

```text
VITE_API_BASE_URL=http://localhost:9000
VITE_APP_TITLE=Agent SkillOps Platform
```

## 六、构建部署

开发阶段：

```bash
pnpm install
pnpm dev
```

生产阶段：

```bash
pnpm build
```

部署方式：

```text
Nginx 托管前端静态资源
gateway-service 提供 API
```
