# 04. 数据库设计

## 一、设计说明

MySQL 作为平台核心元数据存储。

存储边界：

- MySQL：用户、团队、项目、经验、Skill、审核、版本、产物元信息
- FastDFS：附件、生成的 Skill 包、报告文件
- Elasticsearch：检索数据
- Redis：缓存、锁、短期状态

## 二、表设计规范

- 表名使用小写下划线。
- 主键对外使用字符串 ID。
- 所有核心表包含 `created_at`、`updated_at`。
- 审计类数据不做物理删除。
- 发布后的 Skill 版本不可修改。

## 四、Redis 缓存设计

| Key | 说明 | TTL |
| --- | --- | --- |
| `session:{token}` | 登录会话 | 24h |
| `analysis:job:{id}` | 分析任务实时状态 | 24h |
| `draft:lock:{id}` | 草稿编辑锁 | 10m |
| `rate:user:{id}` | 用户限流 | 1m |
| `search:skills:{hash}` | Skill 搜索缓存 | 5m |

### 5.3 reviews_index
