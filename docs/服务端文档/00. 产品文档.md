# 00. 产品文档

## 一、需求背景

随着 Codex、Cursor、Claude Code、LangGraph Agent、自研 Agent 等工具进入研发流程，研发人员每天会通过 Agent 完成大量实际工作：

- 排查环境和部署问题
- 总结项目架构和模块职责
- 生成接口文档、测试方案、代码审查意见
- 沉淀某类任务的执行步骤
- 生成 Codex Skill 草稿
- 提炼某个项目的开发规范或排错经验

但是这些经验通常散落在个人聊天记录、临时 Markdown、PR 评论、本地文件中。个人可以让 Codex 直接生成 Skill，但团队仍然缺少统一平台来回答这些问题：

- 这个 Skill 是否准确？
- 是否泄露了密钥、内网地址、客户信息或私有上下文？
- 是否和已有 Skill 重复？
- 适用于哪个项目、哪个版本、哪类任务？
- 谁审核过？审核意见是什么？
- 后续项目变更后，这个 Skill 是否已经过期？
- 团队成员如何搜索、安装、升级、回滚？

因此，本项目要做的是一个面向研发团队的 **SkillOps 平台**：把个人使用 Agent/Codex 产生的经验、总结、Skill 草稿和相关产出，经过自动分析、人工审核、版本管理和发布流程，沉淀为团队可复用的技能资产。

## 二、产品目标

本平台不尝试替代 Codex，也不做一个更强的 Agent。

平台目标是管理 Agent 周边资产：

1. 收集个人 Agent 工作经验。
2. 规范化生成 Skill 草稿。
3. 对 Skill 草稿进行安全、重复、范围、格式检查。
4. 支持技术负责人审核。
5. 发布团队级 Skill。
6. 提供 Skill 搜索、版本管理、安装包生成。
7. 对 Skill 使用效果进行评价。
8. 对可能过期的 Skill 进行提醒和维护。

## 三、用户角色

| 角色 | 说明 | 主要权限 |
| --- | --- | --- |
| 普通开发者 | 提交经验、Skill 草稿、Agent 产出 | 创建经验记录、查看本人提交、响应修改意见 |
| 技术审核人 | 对 Skill 草稿进行技术审核 | 审核、驳回、要求修改、批准 |
| Skill 维护者 | 维护团队 Skill 仓库 | 发布版本、废弃 Skill、合并重复 Skill |
| 管理员 | 管理团队、项目、用户和策略 | 用户管理、权限配置、平台策略配置 |

## 四、需求范围

### 4.1 一期范围

- 项目管理
- 经验记录提交
- Skill 草稿管理
- 自动分析任务
- 技术审核流程
- Skill 发布和版本管理
- 文件/产物上传与存储
- Skill 搜索
- 基础权限控制

### 4.2 暂不实现

- 替代 Codex/Cursor 直接执行编码任务
- 完全自动发布 Skill
- 企业 SSO
- 插件市场收费
- IDE 插件
- 自动同步到所有开发者本机

## 五、功能描述

### 5.1 首页/工作台

展示当前团队的整体情况：

- 待审核 Skill 数量
- 最近提交的经验记录
- 最近发布的 Skill
- 高风险检查项
- 可能过期的 Skill
- 使用频率较高的 Skill

### 5.2 经验提交

开发者提交一次有价值的 Agent 工作经验。

支持内容：

- 任务标题
- 所属项目
- 来源类型：Codex、Cursor、LangGraph、自研 Agent、人工总结
- 经验总结
- Agent 输出
- 相关文件
- 可选 Skill 草稿

### 5.3 Skill 草稿

平台基于经验记录生成或接收 Skill 草稿。

草稿内容包括：

- Skill 名称
- 触发描述
- `SKILL.md` 内容
- references 文件
- scripts 脚本
- 风险检查报告
- 相似 Skill 推荐

### 5.4 审核中心

技术审核人对 Skill 草稿进行审核。

审核动作：

- 通过
- 驳回
- 要求修改
- 添加评论
- 标记重复
- 标记高风险

### 5.5 Skill 仓库

团队成员可以搜索和查看已发布 Skill。

支持：

- 按项目搜索
- 按任务类型搜索
- 查看版本
- 下载安装包
- 查看审核记录
- 查看使用反馈

### 5.6 Skill 维护

Skill 发布后仍然需要维护。

维护动作：

- 发布新版本
- 废弃旧版本
- 合并重复 Skill
- 标记过期
- 回滚到旧版本

## 六、产品边界

本平台关注的是“团队经验资产化”，而不是“单次 Agent 执行”。

可以类比：

- GitHub 不替开发者写代码，但管理代码协作。
- Confluence 不替员工思考，但管理团队知识。
- 本平台不替 Codex 编程，但管理 Codex/Agent 产生的团队技能资产。
