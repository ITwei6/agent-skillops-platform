# 01. 业务分析

## 一、业务主线

平台核心业务链路如下：

```text
经验提交 -> 自动分析 -> Skill 草稿 -> 技术审核 -> 发布版本 -> 团队复用 -> 反馈维护
```

核心对象不是普通文章，也不是普通 Agent 对话，而是能够被团队复用的 **Skill 资产**。

## 二、页面功能点分析

### 2.1 工作台

功能点：

- 查看待审核 Skill
- 查看本人提交记录
- 查看最近发布 Skill
- 查看高风险草稿
- 查看可能过期 Skill
- 查看平台统计

统计项：

- 本周新增经验数
- 本周新增 Skill 草稿数
- 待审核数量
- 发布数量
- 驳回数量
- 高风险检查数量

### 2.2 项目管理页

功能点：

- 创建项目
- 修改项目基础信息
- 关联代码仓库地址
- 配置项目审核策略
- 查看项目下的经验、Skill、文件

项目是 Skill 归属和权限隔离的重要边界。

### 2.3 经验提交页

功能点：

- 选择项目
- 填写任务标题
- 选择来源类型
- 粘贴 Agent 总结或人工总结
- 上传附件
- 粘贴已有 Skill 草稿
- 提交后触发自动分析任务

来源类型：

- Codex 会话
- Cursor 会话
- Claude Code 会话
- LangGraph 工作流
- PR/Issue 总结
- 人工总结
- 其他

### 2.4 经验详情页

功能点：

- 查看经验原文
- 查看上传附件
- 查看自动分析状态
- 查看生成的 Skill 草稿
- 查看关联的审核记录

### 2.5 Skill 草稿详情页

功能点：

- 查看 `SKILL.md`
- 查看 references/scripts 文件
- 查看自动检查报告
- 查看相似 Skill
- 编辑草稿
- 提交审核
- 查看状态流转历史

### 2.6 审核队列页

功能点：

- 查看待审核草稿
- 按项目、风险等级、提交人、时间过滤
- 认领审核任务
- 批量查看自动检查结果

### 2.7 审核详情页

功能点：

- 查看 Skill 草稿
- 查看来源经验
- 查看自动分析报告
- 添加审核意见
- 通过、驳回、要求修改
- 确认风险项是否已处理

### 2.8 Skill 仓库页

功能点：

- 搜索已发布 Skill
- 按项目、标签、任务类型过滤
- 查看最新版本
- 查看安装说明
- 下载 Skill 包
- 提交使用反馈

### 2.9 Skill 版本页

功能点：

- 查看版本历史
- 查看每个版本的 `SKILL.md`
- 查看版本变更说明
- 查看发布人和发布时间
- 下载历史版本
- 废弃版本

### 2.10 系统管理页

功能点：

- 用户管理
- 团队管理
- 角色权限管理
- 审核策略配置
- 敏感词/密钥检测策略配置
- Agent 分析策略配置

## 四、业务风险

### 4.1 敏感信息泄露

经验记录可能包含：

- token
- 密钥
- 内网地址
- 客户信息
- 私有仓库路径
- 账号密码

平台必须在自动分析阶段进行风险检查。

### 4.2 Skill 触发范围过宽

如果 Skill 描述过宽，Codex 可能在不合适的任务中使用它。

例如：

```text
Use this skill when doing backend development.
```

这种描述过泛，应要求修改为具体场景。

### 4.3 重复建设

多人可能提交类似 Skill。

平台需要在提交和审核阶段提示相似 Skill。

### 4.4 过期风险

项目结构、部署方式、工具版本变化后，旧 Skill 可能误导团队成员。

平台需要支持过期标记和复审提醒。
