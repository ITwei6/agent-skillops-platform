# 11. 过期检测与维护机制

## 一、文档目的

Skill 发布后并不是永久正确。

项目结构、依赖版本、部署方式、工具链、团队规范变化后，旧 Skill 可能误导团队成员。

因此平台必须支持 Skill 过期检测、复审、废弃、更新和回滚机制。

## 二、过期状态

Published Skill 增加维护状态：

```text
fresh
possibly_outdated
outdated
deprecated
superseded
```

说明：

| 状态 | 说明 |
| --- | --- |
| fresh | 当前有效 |
| possibly_outdated | 可能过期，需要复审 |
| outdated | 已确认过期 |
| deprecated | 已废弃，不建议使用 |
| superseded | 被新 Skill 或新版本替代 |

## 三、过期来源

### 3.1 关联文档变化

如果 Skill references 中关联的文档发生变化，应触发复审。

示例：

- 部署文档更新
- 接口文档更新
- 数据库设计更新
- 项目 README 更新

### 3.2 关联代码变化

如果 Skill 关联了具体代码目录或配置文件，相关路径变更后应触发复审。

示例：

- Docker 配置变化
- CMake 构建方式变化
- 服务目录重构
- 脚本入口变化

### 3.3 工具版本变化

示例：

- brpc 版本变化
- RabbitMQ 配置变化
- Elasticsearch 版本变化
- Codex Skill 规范变化

### 3.4 用户反馈

用户反馈以下情况时，应标记为可能过期：

- 按 Skill 操作失败
- 输出结果不符合预期
- 描述与当前项目不一致
- 关键文件不存在

### 3.5 人工标记

reviewer 或 maintainer 可手动标记过期。

## 四、过期检测流程

```text
变更事件产生
  -> 找到关联 Skill
  -> 创建 outdated_check 任务
  -> analysis-worker 分析影响范围
  -> 标记 possibly_outdated
  -> 通知维护者复审
  -> 维护者确认
      -> 保持 fresh
      -> 发布新版本
      -> 标记 outdated
      -> deprecated
```

## 五、关联关系设计

Skill 版本应记录关联对象：

```text
关联项目
关联文档
关联文件路径
关联代码目录
关联服务
关联中间件
关联其他 Skill
```

这些关联关系用于过期检测。

## 六、维护动作

### 6.1 发布新版本

适用：

- Skill 仍有价值，但内容需要更新

要求：

- 创建新版本
- 填写 changelog
- 保留旧版本

### 6.2 标记 deprecated

适用：

- Skill 已不建议使用
- 有替代方案
- 内容长期无人维护

### 6.3 标记 superseded

适用：

- 已被新 Skill 替代

要求：

- 记录替代 Skill ID
- 搜索结果中提示跳转

### 6.4 回滚版本

适用：

- 新版本发布后发现错误

要求：

- 不删除错误版本
- 标记错误版本状态
- 将 latest_version 指回稳定版本
- 记录回滚原因

## 七、复审规则

不同风险等级复审要求不同：

| 风险等级 | 复审要求 |
| --- | --- |
| low | maintainer 确认即可 |
| medium | reviewer 复审 |
| high | reviewer + maintainer 复审 |
| blocked | 禁止使用，必须修复 |

## 八、过期检测消息

Routing Key：

```text
analysis.outdated_check
```

Payload：

```json
{
  "job_id": "job_101",
  "skill_id": "skill_001",
  "version": "1.0.0",
  "change_type": "document_changed",
  "changed_refs": ["docs/服务端文档/02. 项目设计.md"]
}
```

## 九、用户侧展示

Skill 仓库中应展示：

- 当前状态
- 最近更新时间
- 是否可能过期
- 替代 Skill
- 最近反馈
- 维护者

当用户下载或安装可能过期的 Skill 时，平台应提示风险。

## 十、维护指标

平台应统计：

- active Skill 数量
- possibly_outdated 数量
- deprecated 数量
- 平均复审时间
- 过期后仍被下载次数
- 用户反馈触发的过期数量

这些指标用于衡量团队知识资产健康度。
