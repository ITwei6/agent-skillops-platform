# 03. 接口设计

## 一、接口风格

平台对外通过 `gateway-service` 提供 HTTP + JSON API。

内部服务通过 brpc + protobuf 通信。本文先定义对外 HTTP 接口，后续根据稳定接口再拆分 protobuf。

## 二、通用结构

### 2.1 通用响应

```json
{
  "code": "OK",
  "message": "success",
  "request_id": "req_001",
  "data": {}
}
```

### 2.2 分页响应

```json
{
  "items": [],
  "page": 1,
  "page_size": 20,
  "total": 100
}
```

### 2.3 枚举信息

用户角色：

```text
developer
reviewer
maintainer
admin
```

Skill 草稿状态：

```text
draft
submitted
needs_changes
approved
rejected
published
```

分析任务状态：

```text
queued
running
succeeded
failed
```

## 三、用户与团队接口

### 3.1 创建用户

```http
POST /api/v1/users
```

请求：

```json
{
  "team_id": "team_001",
  "name": "Alice",
  "email": "alice@example.com",
  "role": "developer"
}
```

响应：

```json
{
  "user_id": "user_001"
}
```

### 3.2 获取当前用户

```http
GET /api/v1/users/me
```

## 四、项目接口

### 4.1 创建项目

```http
POST /api/v1/projects
```

请求：

```json
{
  "team_id": "team_001",
  "name": "agent-skillops-platform",
  "description": "团队 SkillOps 平台",
  "repo_url": "git@github.com:ITwei6/agent-skillops-platform.git"
}
```

### 4.2 获取项目列表

```http
GET /api/v1/projects?page=1&page_size=20
```

### 4.3 获取项目详情

```http
GET /api/v1/projects/{project_id}
```

## 五、经验记录接口

### 5.1 提交经验记录

```http
POST /api/v1/experiences
```

请求：

```json
{
  "project_id": "proj_001",
  "title": "Docker 开发环境 SSH 连接排查经验",
  "source_type": "codex_session",
  "summary": "排查 Windows VS Code 连接 Ubuntu 虚拟机 Docker 开发容器的问题...",
  "artifact_ids": ["art_001"]
}
```

响应：

```json
{
  "experience_id": "exp_001",
  "analysis_job_id": "job_001"
}
```

### 5.2 查询经验详情

```http
GET /api/v1/experiences/{experience_id}
```

### 5.3 查询经验列表

```http
GET /api/v1/experiences?project_id=proj_001&page=1&page_size=20
```

## 六、Skill 草稿接口

### 6.1 创建 Skill 草稿

```http
POST /api/v1/skill-drafts
```

请求：

```json
{
  "project_id": "proj_001",
  "experience_id": "exp_001",
  "name": "docker-remote-ssh-troubleshooting",
  "description": "用于排查 Docker 开发环境远程 SSH 连接问题。",
  "skill_md": "---\nname: docker-remote-ssh-troubleshooting\n..."
}
```

### 6.2 查询 Skill 草稿

```http
GET /api/v1/skill-drafts/{draft_id}
```

### 6.3 修改 Skill 草稿

```http
PATCH /api/v1/skill-drafts/{draft_id}
```

### 6.4 提交审核

```http
POST /api/v1/skill-drafts/{draft_id}/submit
```

## 七、审核接口

### 7.1 查询审核队列

```http
GET /api/v1/reviews/queue?project_id=proj_001&status=submitted
```

### 7.2 提交审核意见

```http
POST /api/v1/skill-drafts/{draft_id}/reviews
```

请求：

```json
{
  "decision": "needs_changes",
  "comments": "触发描述过宽，请限制到 Docker 开发环境 SSH 排查场景。"
}
```

审核决策：

```text
approve
reject
needs_changes
comment
```

## 八、发布接口

### 8.1 发布 Skill

```http
POST /api/v1/skill-drafts/{draft_id}/publish
```

请求：

```json
{
  "version": "1.0.0",
  "changelog": "初始版本。"
}
```

### 8.2 查询 Skill 仓库

```http
GET /api/v1/skills?query=docker&project_id=proj_001
```

### 8.3 查询 Skill 版本详情

```http
GET /api/v1/skills/{skill_id}/versions/{version}
```

### 8.4 下载 Skill 包

```http
GET /api/v1/skills/{skill_id}/versions/{version}/package
```

## 九、文件产物接口

### 9.1 上传文件

```http
POST /api/v1/artifacts
Content-Type: multipart/form-data
```

字段：

- `project_id`
- `owner_type`
- `owner_id`
- `file`

响应：

```json
{
  "artifact_id": "art_001",
  "sha256": "..."
}
```

### 9.2 下载文件

```http
GET /api/v1/artifacts/{artifact_id}/download
```

## 十、分析任务接口

### 10.1 查询分析任务

```http
GET /api/v1/analysis-jobs/{job_id}
```

### 10.2 重试分析任务

```http
POST /api/v1/analysis-jobs/{job_id}/retry
```
