# 15. 第一阶段服务端工程骨架

## 目标

第一阶段只建立服务端工程骨架，不实现完整业务闭环。

已包含：

- 根 CMake 工程
- C++ 微服务基础库依赖适配
- `services/` 服务目录
- `workers/` Worker 目录
- `idl/` 契约目录
- `deploy/config/` 配置目录
- `gateway-service` 最小 HTTP 服务
- `gateway-service` 到内部服务的最小 HTTP 转发
- `identity-service` 用户创建与当前用户查询接口
- `project-service` 项目创建、列表、详情接口

## 基础库依赖

业务仓库不复制基础库源码，只通过 CMake 包依赖基础库。

当前依赖适配顺序：

1. `microkit` / `microkit::microkit`
2. `tew_scaffold` / `tew_scaffold::tew_scaffold`
3. `bite_scaffold` / `bite_scaffold::bite_scaffold`

其中 `tew_scaffold` 是当前本地 `cpp-microservice-kit` 实际导出的包名和 target 名称。

如果本地宿主机未安装基础库，CMake 会使用标准库 fallback 让第一阶段骨架可配置；正式编译和验证仍应在 `dev-env-service` 容器内执行。

## 容器内编译基础库

```bash
docker exec dev-env-service bash -lc "cd /home/dev/workspace/kits/cpp-microservice-kit && mkdir -p build && cd build && cmake .. -DCMAKE_INSTALL_PREFIX=/usr && make -j\$(nproc) && sudo make install"
```

如果容器实际目录是 `/home/dev/workspace/third_party/cpp-microservice-kit`，则把命令中的路径替换为该目录。

## 容器内编译业务项目

```bash
docker exec dev-env-service bash -lc "cd /home/dev/workspace/projects/agent-skillops-platform && mkdir -p build && cd build && cmake .. && make -j\$(nproc)"
```

## 验证接口

```bash
curl http://127.0.0.1:8080/healthz
curl http://127.0.0.1:8080/api/v1/ping
curl -X POST http://127.0.0.1:8080/api/v1/users \
  -H "Content-Type: application/json" \
  -d '{"team_id":"team_1","name":"Admin","email":"admin@example.com","role":"owner"}'
curl http://127.0.0.1:8080/api/v1/users/me
curl -X POST http://127.0.0.1:8080/api/v1/projects \
  -H "Content-Type: application/json" \
  -d '{"team_id":"team_1","name":"SkillOps","description":"Team skill governance","repo_url":"https://example.com/repo.git","created_by":"user_1"}'
curl http://127.0.0.1:8080/api/v1/projects
curl http://127.0.0.1:8080/api/v1/projects/proj_1
curl -X POST http://127.0.0.1:8080/api/v1/experiences \
  -H "Content-Type: application/json" \
  -d '{"project_id":"proj_1","title":"Docker SSH troubleshooting","source_type":"codex_session","summary":"Troubleshoot Docker development container access.","artifact_ids":["art_1"]}'
curl "http://127.0.0.1:8080/api/v1/experiences?project_id=proj_1&page=1&page_size=20"
curl http://127.0.0.1:8080/api/v1/experiences/exp_1
curl -X POST http://127.0.0.1:8080/api/v1/skill-drafts \
  -H "Content-Type: application/json" \
  -d '{"project_id":"proj_1","experience_id":"exp_1","name":"docker-ssh-troubleshooting","description":"Troubleshoot Docker SSH access.","skill_md":"name: docker-ssh-troubleshooting\nworkflow: check ssh and docker ports"}'
curl http://127.0.0.1:8080/api/v1/skill-drafts/draft_1
curl -X POST http://127.0.0.1:8080/api/v1/skill-drafts/draft_1/submit
curl -X POST http://127.0.0.1:8080/api/v1/skill-drafts/draft_1/publish \
  -H "Content-Type: application/json" \
  -d '{"version":"1.0.0","changelog":"Initial version"}'
curl "http://127.0.0.1:8080/api/v1/skills?query=docker&project_id=proj_1"
curl -X POST http://127.0.0.1:8080/api/v1/skill-drafts/draft_1/reviews \
  -H "Content-Type: application/json" \
  -d '{"project_id":"proj_1","reviewer_id":"user_1","decision":"needs_changes","comments":"Tighten trigger scope."}'
curl "http://127.0.0.1:8080/api/v1/reviews/queue?project_id=proj_1&status=needs_changes"
curl -X POST http://127.0.0.1:8080/api/v1/artifacts \
  -H "Content-Type: application/json" \
  -d '{"project_id":"proj_1","owner_type":"experience","owner_id":"exp_1","file_name":"session-summary.md","content_type":"text/markdown","sha256":"abc123","size_bytes":128}'
curl "http://127.0.0.1:8080/api/v1/artifacts?project_id=proj_1&owner_type=experience&owner_id=exp_1"
curl http://127.0.0.1:8080/api/v1/artifacts/art_1
curl http://127.0.0.1:8080/api/v1/artifacts/art_1/download
```

预期返回：

```json
{"code":"OK","message":"success","request_id":"req_1","data":{}}
```
