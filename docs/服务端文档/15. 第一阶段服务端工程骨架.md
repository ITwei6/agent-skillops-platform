# 15. 第一阶段服务端工程骨架

## 目标

第一阶段只建立服务端工程骨架，不实现完整业务闭环。

已包含：

- 根 CMake 工程
- C++ 微服务基础库依赖适配
- `services/` 服务目录
- `workers/` Worker 目录
- `idl/` 契约目录
- `deploy/config/` 配置目录
- `gateway-service` 最小 HTTP 服务
- `identity-service` 和 `project-service` 最小可执行骨架

## 基础库依赖

业务仓库不复制基础库源码，只通过 CMake 包依赖基础库。

当前依赖适配顺序：

1. `microkit` / `microkit::microkit`
2. `tew_scaffold` / `tew_scaffold::tew_scaffold`
3. `bite_scaffold` / `bite_scaffold::bite_scaffold`

其中 `tew_scaffold` 是当前本地 `cpp-microservice-kit` 实际导出的包名和 target 名称。

## 容器内编译基础库

```bash
docker exec dev-env-service bash -lc "cd /home/dev/workspace/kits/cpp-microservice-kit && mkdir -p build && cd build && cmake .. -DCMAKE_INSTALL_PREFIX=/usr && make -j\$(nproc) && sudo make install"
```

如果容器实际目录是 `/home/dev/workspace/third_party/cpp-microservice-kit`，则把命令中的路径替换为该目录。

## 容器内编译业务项目

```bash
docker exec dev-env-service bash -lc "cd /home/dev/workspace/projects/agent-skillops-platform && mkdir -p build && cd build && cmake .. && make -j\$(nproc)"
```

## 运行 gateway-service

```bash
docker exec dev-env-service bash -lc "cd /home/dev/workspace/projects/agent-skillops-platform && ./build/services/gateway-service/gateway-service --host=0.0.0.0 --port=8080"
```

## 验证接口

```bash
curl http://127.0.0.1:8080/healthz
curl http://127.0.0.1:8080/api/v1/ping
```

预期返回：

```json
{"code":"OK","message":"success","request_id":"req_1","data":{}}
```
