# 05. 消息队列设计

## 一、设计目标

消息队列用于处理耗时、可重试、易失败、不适合同步阻塞的任务。

本平台主要异步任务：

- 根据经验生成 Skill 草稿
- 对 Skill 草稿进行自动检查
- 生成 Skill 安装包
- 写入搜索索引
- 发送审核通知
- 检测可能过期的 Skill

## 二、Exchange 设计

| Exchange | 类型 | 说明 |
| --- | --- | --- |
| `skillops.analysis` | topic | 自动分析任务 |
| `skillops.package` | topic | Skill 打包任务 |
| `skillops.index` | topic | 搜索索引任务 |
| `skillops.notification` | topic | 通知任务 |
| `skillops.media` | topic | 多媒体经验资产处理 |
| `skillops.deadletter` | topic | 死信消息 |

## 五、消息定义

### 5.1 生成 Skill 草稿

Routing Key：

```text
analysis.generate_skill
```

Payload：

```json
{
  "job_id": "job_001",
  "experience_id": "exp_001",
  "project_id": "proj_001",
  "artifact_ids": ["art_001"],
  "requested_by": "user_001"
}
```

处理结果：

- 创建或更新 Skill 草稿
- 创建自动检查报告
- 更新分析任务状态

### 5.2 检查 Skill 草稿

Routing Key：

```text
analysis.check_skill
```

Payload：

```json
{
  "job_id": "job_002",
  "draft_id": "draft_001",
  "check_types": ["secret", "duplicate", "scope", "format"]
}
```

检查项：

- 是否包含敏感信息
- 是否和已有 Skill 重复
- 触发描述是否过宽
- `SKILL.md` 结构是否合规
- 是否包含危险脚本或危险命令

### 5.3 构建 Skill 包

Routing Key：

```text
package.build_skill
```

Payload：

```json
{
  "job_id": "job_003",
  "skill_version_id": "sv_001",
  "skill_id": "skill_001",
  "version": "1.0.0"
}
```

产物：

- `SKILL.md`
- references/
- scripts/
- metadata.json
- zip/tar.gz 安装包

### 5.4 写入 Skill 索引

Routing Key：

```text
index.skill
```

Payload：

```json
{
  "skill_id": "skill_001",
  "version": "1.0.0",
  "operation": "upsert"
}
```

### 5.5 审核通知

Routing Key：

```text
notification.review
```

Payload：

```json
{
  "draft_id": "draft_001",
  "receiver_user_id": "user_002",
  "notification_type": "review_requested"
}
```

## 六、重试策略

| 任务 | 最大重试次数 |
| --- | --- |
| 生成 Skill 草稿 | 3 |
| 自动检查 | 3 |
| 构建 Skill 包 | 2 |
| 写入索引 | 5 |
| 通知 | 3 |

失败后进入死信队列。

## 七、幂等设计

消费者必须使用 `event_id` 和 `job_id` 做幂等控制。

要求：

- 同一个生成任务不能生成多个重复草稿。
- 同一个 Skill 版本不能重复发布。
- 索引写入使用 upsert。
- 通知任务尽量避免重复发送。
