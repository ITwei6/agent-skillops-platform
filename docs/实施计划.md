# 实施计划

## 一、当前阶段定位

当前阶段是项目设计阶段，不进入业务代码实现。

目标是先完成一套可以指导实现窗口编码的服务端文档，类似视频点播系统的服务端课件文档。

## 二、阶段划分

### 阶段 1：需求与架构冻结

交付物：

- 产品文档
- 业务分析
- 项目设计
- 接口设计
- 数据库设计
- 消息队列设计
- 开发规范
- 子服务设计总览

验收标准：

- 项目边界清晰
- 服务拆分清晰
- 核心状态机清晰
- 主要接口清晰
- 数据表初稿清晰
- MQ 任务清晰

### 阶段 2：子服务详细设计

交付物：

- gateway-service 设计文档
- identity-service 设计文档
- project-service 设计文档
- experience-service 设计文档
- skill-service 设计文档
- review-service 设计文档
- artifact-service 设计文档
- analysis-worker 设计文档

验收标准：

- 每个服务职责明确
- 接口、数据、缓存、MQ、错误码完整
- 可以直接交给实现窗口编码

### 阶段 3：基础工程搭建

由实现窗口负责。

建议顺序：

1. 初始化 C++ 工程结构
2. 接入脚手架
3. 搭 gateway-service
4. 搭 identity-service
5. 搭 project-service

### 阶段 4：核心业务闭环

由实现窗口负责。

目标：

```text
项目 -> 用户 -> 经验 -> Skill 草稿 -> 审核 -> 发布
```

### 阶段 5：Agent/RAG 增强

由实现窗口负责。

目标：

- analysis-worker
- 自动生成 Skill 草稿
- 重复检查
- 敏感信息检查
- 触发范围检查

## 三、实现窗口交接方式

实现窗口每次开始编码前，应先阅读：

1. `AGENTS.md`
2. `docs/服务端文档/`
3. 对应 `docs/子服务文档/`

如果实现发现设计不合理，先反馈设计窗口，不直接改业务方向。

## 四、窗口分工

### 4.1 当前窗口

职责：

- 产品大纲
- 技术架构
- 服务端文档
- 前端文档
- 子服务设计
- 迭代规划
- 验收标准

不负责：

- 直接编写业务代码
- 临时决定接口字段
- 绕过文档修改方向

### 4.2 窗口 A：后端实现

职责：

- C++ 后端工程搭建
- gateway-service
- 各 C++ 子服务
- 数据库访问
- MQ 接入
- 服务注册发现

实现依据：

- `docs/服务端文档/`
- `docs/子服务文档/`

### 4.3 窗口 B：前端实现

职责：

- Vue 前端工程搭建
- 页面实现
- 组件实现
- API 对接
- 前端权限
- 前端交互体验

实现依据：

- `docs/前端文档/`
- `docs/服务端文档/03. 接口设计.md`
